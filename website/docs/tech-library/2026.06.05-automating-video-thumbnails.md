I had a way to create thumbnails for the videos that made in the past. I wanted to start 
that process back up, now that I have made changes to the video automation script. How can 
this be done? 

I used my static website to make this possible. 

Now the thumbnails will not be fancy like you see many of creators do.  Yes I could leverage
AI to come up with a thumbnail to be used, but I am not really feeling the AI overload and 
forcing of AI that the big players are doing. 

Instead I created a single page on my static website and utilized some Javascript. 

The Javascript code looks at the query parameters from the URL to determine what 
should be changed on the page and the text that should be printed on screen. Below 
is the current version of the JavaScript and HTML. 

## Javascript Code

```js
const sideBarColumn = document.getElementById("sidebar");
const titleColumn = document.getElementById("title");

const params = new URLSearchParams(document.location.search);
const video = params.get("video") ?? null;
const title = params.get("title") ?? null;

function lightSidebar() {
sideBarColumn.classList.add("bg-light");
titleColumn.classList.add("text-light");
}

function darkSidebar() {
sideBarColumn.classList.add("bg-dark");
titleColumn.classList.add("text-dark");
}

if (title !== null) {
titleColumn.innerText = title.toUpperCase();
}

switch (video) {
case "gardening":
lightSidebar();
titleColumn.classList.add("bg-success");
break;

case "handyman":
darkSidebar();
titleColumn.classList.add("bg-warning");
break;

case "personal":
darkSidebar();
titleColumn.classList.add("bg-info");
break;

case "tech":
darkSidebar();
titleColumn.classList.add("bg-success-subtle");
break;

default:
darkSidebar();
titleColumn.classList.add("bg-light");
break;
}
```

As you see with the web page, it is a very minimal web page.  The formatting and classes 
on the page that will not change regardless of the thumbnail, are coded into the page. 

Then using the javascript, the colors of the page are changed and the text of the page 
is populated into the specific div.  


Next part of this will require that a Selenium Webdriver, or similar automation, go to the URL 
with the query parameters included.  Then take a snapshot of the page and save the snapshot 
to disk.  

FFMPEG will then use that snapshot, to create the end screen for the video and render the video.
After uploading the video, the thumbnail can be selected. Because the thumbnail is at the end of the 
video, everything is uploaded at once and reduces the number of files that have to be managed.

This set up is designed so that everything is part of the website, but if it is 
really needed, I can generate the static files and run the website locally without 
many changes.
